IntersectionObserver 是浏览器原生提供的一个观察器 API，用来 监测某个目标元素与可视区域或某个其他元素交叉状态 的变化，例如：是否进入视口、进入多少。
它可以取代传统的 scroll + getBoundingClientRect() 实现懒加载、无限滚动、曝光统计等功能，性能更高，因为它是浏览器异步回调和内部优化处理的。

场景常用在哪？

- 懒加载图片（图片快滑到可视区域再加载）
- 页面元素曝光埋点（广告曝光、统计进入视口次数）
- 无限滚动加载更多
- 触发动画（元素进入屏幕时播放动画）
- Sticky 组件、吸顶相关计算

| 方案                             | 是否节流       | 是否依赖滚动事件 | 浏览器优化级别 |
|----------------------------------|-------------|---------------|--------------|
| `scroll + getBoundingClientRect` | 需手动函数节流 | 重度依赖         | 中等           |
| `IntersectionObserver`           | 不需要         | 不依赖           | 高（异步批处理） |


| 特点     | 描述                               |
|--------|------------------------------------|
| 核心功能 | 监听元素是否进入某个可视区域       |
| 优点     | 性能好、无需监听滚动、浏览器原生支持 |
| 参数控制 | root, threshold, rootMargin        |
| 用途场景 | 懒加载、曝光统计、动画触发、无限滚动  |
